# [專題名稱] 平台

## Proposal Report

### 動機與目標
動機：普通手機內建的行事曆只能顯示行程時間或任務截止日期，不具有協助安排工作進度的功能。<br>
目標：建立簡單的圖形化介面，將任務以「彩色方塊」顯示。並透過緊急、重要程度，幫助使用者規劃每天的工作進度。

### 預期功能
任務分類：每個任務是一個有顏色的矩形，顯示簡單的任務名稱。使用者可以為不同類別的任務選擇不同的底色。
自動排序：系統會根據截止日期（Deadline）、使用者期望花費多少時間、使用者當天有多少工作時間，自動排列方塊，規劃使用者每天應完成的工作。
簡易互動：點擊方塊代表「完成」，方塊會消失（從資料結構中移除）。

### 使用技術
程式語言：C++ 20
圖形庫：SFML 3.0
開發工具：VS Code

### 與課程的關聯
Priority Queue (優先權佇列)：使用 std::priority_queue 來存放任務，並自定義比較函數（Comparator），讓時間最接近的任務排在最前面。

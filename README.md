# 任務規劃調色盤

## Proposal Report

### 動機與目標
**動機**：多數手機內建行事曆僅具備「記錄」功能，無法主動協助使用者平衡每日工作量，常導致截止日前夕工作堆積。<br>
**目標**：建立視覺直觀的圖形化介面，將任務轉化為「彩色方塊」。系統將結合任務的緊急程度與所需時間，自動生成每日工作建議，協助使用者達成高效的時間規劃。<br>

### 預期功能
**任務分類與視覺化**：每個任務以彩色矩形呈現，使用者可依類別（如課業、社團、生活）自定義底色，快速辨識任務屬性。<br>
**智能排程邏輯**：系統綜合「截止日期」、「預計工時」與「每日可用時間」，自動計算優先級並排列方塊，產出每日最優化工作順序。<br>
**動態互動機制**：點擊方塊即標記「完成」，該物件會從底層資料結構中移除，畫面同步更新。<br>

### 使用技術
**程式語言**：C++ 20<br>
**圖形庫**：SFML 3.0<br>
**開發環境**：VS Code + MSYS2 (UCRT64)<br>

### 時程規劃
**第 六　 週**：架設環境與初步構想（已完成）<br>
**第 七　 週**：完成提案報告與資料結構設計計畫<br>
**第 八　 週**：期中考週（進度暫停）<br>
**第 九　 週**：定義 Task 結構與自定義比較運算子<br>
**第 十　 週**：實作基於 Priority Queue 的任務排序與工時分配算法<br>
**第 十一 週**：開發 SFML 基礎 UI 介面、Prototype Report<br>
**第 十二 週**：實作滑鼠監聽與任務狀態變更邏輯<br>
**第 十三 週**：介面優化（顏色過渡效果與佈局美化）<br>
**第 十四 週**：Final Report

### 與課程的關聯
**Priority Queue (優先權佇列)**：透過 std::priority_queue 存儲 Task 物件，並實作自定義 Comparator，依據截止日期與工時權重進行動態排序。<br>
**Struct & Vector**：利用 struct 封裝任務屬性，並使用 std::vector 管理當前顯示於畫面上的 UI 物件陣列，實踐資料與視覺的分離管理。<br>

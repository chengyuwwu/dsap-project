# 任務規劃調色盤

## Proposal Report

### 動機與目標
*   **動機**：多數手機內建行事曆僅具備「記錄」功能，無法主動協助使用者平衡每日工作量，常導致截止日前夕工作堆積。<br>
*   **目標**：建立視覺直觀的圖形化介面，將任務轉化為「彩色方塊」。系統將結合任務的緊急程度與所需時間，自動生成每日工作建議，協助使用者達成高效的時間規劃。<br>

### 競品比較
| 軟體名稱 | 主要功能 | 本專案優勢 |
| :--- | :--- | :--- |
| **Google Calendar** | 傳統格子式日曆，側重行程記錄 | **視覺化排程**：以彩色方塊直觀呈現剩餘工時，非單純文字 |
| **Microsoft To Do** | 條列式清單，手動排序 | **智能分配**：根據每日可用時間，自動計算並建議今日工作量 |
| **Notion** | 強大的資料庫與筆記功能 | **輕量與專注**：專注於「時間壓力視覺化」，無須複雜設定 |

### 預期功能
*   **任務分類與視覺化**：每個任務以彩色矩形呈現，使用者可依類別（如課業、社團、生活）自定義底色，快速辨識任務屬性。<br>
*   **智能排程邏輯**：系統綜合「截止日期」、「預計工時」與「每日可用時間」，自動計算優先級並排列方塊，產出每日最優化工作順序。<br>
*   **動態互動機制**：點擊方塊即標記「完成」，該物件會從底層資料結構中移除，畫面同步更新。<br>

### 使用技術
*   **程式語言**：C++ 20<br>
*   **圖形庫**：SFML 3.0<br>
*   **開發環境**：VS Code + MSYS2 (UCRT64)<br>

### Prototype 預計可驗證內容
*   **資料結構驗證**：驗證 std::priority_queue 能否正確依照自定義權重（截止日期+工時）排序任務。<br>
*   **圖形渲染驗證**：在 SFML 視窗中根據資料結構內容，動態生成對應數量的彩色矩形。<br>
*   **基礎互動驗證**：測試滑鼠點擊矩形後，能否成功觸發資料刪除並重新渲染畫面。<br>

## Prototype Report

### 目前進度
*   **使用者介面 (UI) 建立**：使用 SFML 實作互動式視窗，包含任務列表、編輯選單與輸入對話框。<br>
*   **簡易任務排程**：初步實現基於截止日期（Deadline）與預估工時（EST）的優先權排序。<br>
*   **資料持久化**：支援 CSV 檔案讀取與儲存，實現任務資料與使用者工作時數（Daily Capacity）的跨Session保存。<br>
*   **動態狀態視覺化**：根據任務日期自動判定並顯示顏色（今日：紅色、明日：粉色、過期：灰色）。<br>

### 遇到的困難
*   **效能分析的資料規模問題**<br>
    *   **困難**：目前任務量較小，傳統 $O(N \log N)$ 與 $O(\log N)$ 演算法的執行時間差異無法透過體感觀測。<br>
    *   **對策**：計畫撰寫測試腳本模擬 1,000 ~ 10,000 筆任務的動態操作，量化不同資料結構的效能臨界點。<br>
*   **動態權重導致的頻繁排序開銷**<br>
    *   **困難**：本專案引入「建議每日工時」權重，該數值隨日期流逝而變動，導致排序必須定期全體刷新。<br>
    *   **對策**：重新評估 `std::vector` 的連續記憶體優勢在頻繁 `std::sort` 下，是否比重建 `std::priority_queue` 更有利於 UI 渲染。<br>
*   **工作建議邏輯的邊界處理**<br>
    *   **困難**：當任務今日截止時，平均工時計算會面臨分母為零或數值異常的問題。<br>
    *   **對策**：實作條件分支法，強制將今日截止任務設為「絕對優先」，建議工時直接等於剩餘總工時。<br>
*   **狀態保存的一致性**<br>
    *   **困難**：原先 Work Hour 設定無法在程式重開後保留。<br>
    *   **對策**：重構 `Scheduler` 類別，將設定值整合至 CSV 持久化流程中。<br>

### 下一步計畫
*   **短期目標 (Short-term)**<br>
    *   **工時預警系統**：根據「建議工時 / Daily Capacity」比例實作綠、黃、紅三色警示。<br>
    *   **邏輯封裝**：將排序與權重算法完全移至 `Scheduler` 類別，優化程式碼結構。<br>
*   **中期目標 (Mid-term)**<br>
    *   **任務分類與篩選**：支援 Tag 功能（學業、工作、生活）並實作 UI 篩選器。<br>
    *   **輸入檢查優化**：強化日期格式攔截，防止不合規數據導致程式崩潰。<br>
*  **長期目標 (Long-term)**<br>
    *   **遊戲化機制 (Gamification)**：建立 XP 經驗值系統，完成任務可升級；任務逾期則扣除體力值。<br>
    *   **數據趨勢圖表**：實作視覺化統計，回顧每週任務完成效率。<br>
